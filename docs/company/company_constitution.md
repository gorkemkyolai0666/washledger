# Product Principles

- Products must be web-based.
- Mobile applications are not allowed unless absolutely necessary.
- Products must run in modern browsers without installation.

- Existing market categories are allowed if clear innovation exists.
- Avoid direct clones.

- Products must be feasible for a medium-sized team.

- Products must be suitable for parallel development by multiple agents.

- Monetization must be clearly defined for:

  - short term
  - medium term
  - long term

- Study successful competitors' monetization models before deciding pricing.

- Every project must have a sustainable business model.

- Every idea must pass market validation before implementation.

- Building the product must be more valuable than not building it.

# Company Constitution

Documentation First.

Research Before Coding.

Every Feature Requires Documentation.

Every Feature Requires Testing.

Every Architecture Decision Must Be Logged.

Every Task Must Have Acceptance Criteria.

Project Knowledge Must Survive Context Loss.

Avoid Quick Fixes.

Maintain High Code Quality.

Always Update Documentation.

# Language Policy

Primary language: Turkish

Requirements:

- All user-facing text must be Turkish.
- All UI labels must be Turkish.
- All buttons must be Turkish.
- All validation messages must be Turkish.
- All notifications must be Turkish.
- All email templates must be Turkish.
- All documentation must be Turkish.
- All PRD documents must be Turkish.
- All roadmap documents must be Turkish.
- All architecture documents must be Turkish.

Code rules:

- Source code remains in English.
- Variable names remain in English.
- Class names remain in English.
- Method names remain in English.
- Database table names remain in English.

Only user-facing content should be Turkish.

# Documentation Language Policy

The target audience for project documentation is Turkish.

Therefore:

* All project documentation must be written in Turkish.
* All research documents must be written in Turkish.
* All PRD documents must be written in Turkish.
* All roadmap documents must be written in Turkish.
* All architecture explanations must be written in Turkish.
* All market research reports must be written in Turkish.
* All competitor analysis documents must be written in Turkish.
* All final project reports must be written in Turkish.

The following files must always be maintained in Turkish:

* PRD.md
* ROADMAP.md
* MARKET_RESEARCH.md
* COMPETITOR_ANALYSIS.md
* USER_PAIN_POINTS.md
* MONETIZATION.md
* MVP_VALIDATION.md
* FINAL_DOCUMENTATION.md

Code should remain in English.

Variable names, class names, method names, database entities and technical identifiers must remain in English.
